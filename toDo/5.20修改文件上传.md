## 8. Upload

上传接口用于大文件分片上传。当前实现已经做了用户归属校验、chunk index 校验、chunk size 校验、重复分片计数保护、hash/size 合并校验。

### 8.1 初始化上传

```http
POST /api/v1/upload/init
Authorization: Bearer <token>
Content-Type: application/json
```

请求体:

```diff
{
  "filename": "input_video.mp4",
  "file_size": 10485760,
  "chunk_size": 5242880,
  
+ "mime_type": "video/mp4", //用于表示文件类型，具体按照下面表格来
+ "file_hash": "sha256-string",//文件哈希确认，防止上传重复，同时防止后续请求出现问题

}
```
mime_type值文件说明
| mime_type值 | 子类 | 说明 |
| --- | --- | --- |
|  video | mp4 mov| 视频文件 |
| image |  jpg / jpeg png| 图片 |
| model | ply | 3dgs模型 |
| other | zip json | 可能的其他文件，比如相机位姿之类的 |


响应:

```diff
{
  "upload_id": "6b0a5f16-4d75-41f2-9f74-1c6b826d72f0",

  "chunk_size": 5242880,
  "total_chunks": 2,
  "expires_at": "2026-05-20T10:00:00"
}
```

### 8.2 上传分片

```diff
- POST /api/v1/upload/{upload_id}/chunk
+ PUT /api/v1/upload/{upload_id}/chunk
Authorization: Bearer <token>
- Content-Type: multipart/form-data
+  Content-Type: application/octet-stream
```

表单字段:

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `chunk_index` | int | 是 | 从 0 开始 |
| `chunk` | file | 是 | 当前分片二进制 |

响应:

```diff
{
  "received": true,
  "chunk_index": 0,
+ "etag": "d41d8cd98f00b204e9800998ecf8427e"  // 后端计算出的该分片 MD5 值,用于防止上传丢包
} 
```

### 8.3 查询上传进度

```http
GET /api/v1/upload/{upload_id}/progress
Authorization: Bearer <token>
```

响应:

```diff
{
  "upload_id": "6b0a5f16-4d75-41f2-9f74-1c6b826d72f0",
  "filename": "input_video.mp4",
  "file_size": 10485760,
  "total_chunks": 2,
  "received_chunks": 1,
  "status": "uploading",
  "chunk_statuses": [2, 0],

-  "chunk_status_meaning": {
-    "0": "pending",
-   "1": "uploading",
-    "2": "uploaded"
-  } //这个字段没啥用呀

}
```

`chunk_statuses` 用于断点续传，数组下标就是 `chunk_index`:

| 值 | 含义 | 前端动作 |
| --- | --- | --- |
| `0` | 待上传 | 需要继续上传该分片 |
| `1` | 上传中 | 同一 API 进程内正在保存该分片，通常可稍后重查 |
| `2` | 已上传 | 跳过该分片 |

例如 5 个分片返回:

```json
[0, 2, 2, 0, 2]
```

表示第 1、4 个分片还需要上传，第 2、3、5 个分片已经上传。当前服务端稳定持久化的是 `0/2`，`1` 是短暂状态，主要用于同一进程内并发上传时观察。

### 8.4 合并分片

```diff
POST /api/v1/upload/{upload_id}/merge
Authorization: Bearer <token>
- Content-Type: multipart/form-data
+ Content-Type: application/json
```

表单字段:
``` diff
- | 字段 | 类型 | 必填 | 说明 |
- | --- | --- | --- | --- |
- - | `expected_hash` | string | 否 | 完整文件 sha356，传空则不校验 |
- + | `expected_hash` | string | 否 | 完整文件 md5，传空则不校验 |
- | `expected_size` | int | 否 | 完整文件字节数，传 0 则不校验 |
- ```
body
```diff
+ {
+ "expected_hash": "d2d2d2d2...", // 整个大文件的最终 MD5 (可选)
+   "expected_size": 10485760,
+   "parts": [
+     {"chunk_index": 0, "etag": "d41d8cd98f00b204e9800998ecf8427e"},
+     {"chunk_index": 1, "etag": "另一个md5值..."}
+   ]
+ }
```
响应:

```json
{
  "file_id": 12,
  "file_hash": "d2d2d2d2...",
  "storage_key": "8f4f0cc0...",
  "verified": true
}
```

### 8.5 取消上传

```http
POST /api/v1/upload/{file_id}/cancel
Authorization: Bearer <token>
```

响应:

```diff
{
  "cancelled": true,
 "upload_id": "6b0a5f16-4d75-41f2-9f74-1c6b826d72f0"

}
```
